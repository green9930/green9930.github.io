Markdown Syntax

https://heropy.blog/2017/09/30/markdown/

제목(Header)

  • 제목 1

  • 제목 2

  • 제목 3

  • 제목 4

  • 제목 5
  • 제목 6

  • 제목 1 ======

  • 제목 2 ——

강조(Emphasis)

  • Italic : *(asterisk) or _(underscore)
  • Bold : ** or __
    -> 같이 쓰는 경우 _이탤릭체_와 볼드체를 한번에
  • Underline :
  • Strikethrough : ~~(tilde)

목록(List)

    ,
      목록 태그로 변환 1. 순서가 필요한 목록 1. 순서가 필요한 목록 - 순서가 필요하지 않은 목록(서브) - 순서가 필요하지 않은 목록(서브) 1. 순서가 필요한 목록 1. 순서가 필요한 목록(서브) 1. 순서가 필요한 목록(서브) 1. 순서가 필요한 목록
      • 순서가 필요하지 않은 목록에 사용 가능한 기호
        • 대쉬(hyphen)
        • 별표(asterisks)
        • 더하기(plus sign)

      링크(Link)

      로 변환
      GOOGLE

      NAVER

      상대적 참조

      [Dribbble][Dribbble link]

      [GitHub][1]

      문서 안에서 [참조 링크]를 그대로 사용할 수도 있음
      문서 내 일반 URL이나 꺾쇠 괄호(< >, Angle Brackets)안의 URL은 자동으로 링크를 사용합니다.

      • 구글 홈페이지: https://google.com

      • 네이버 홈페이지: https://naver.com

      • [Dribbble link]: https://dribbble.com

      • [1]: https://github.com

      • [참조 링크]: https://naver.com “네이버로 이동합니다!”

      이미지(Image)

      로 변환
      링크와 비슷하나 앞에 !가 붙음

      이미지에 링크

      마크다운 이미지 코드를 링크 코드로 묶어줌

      코드(Code) 강조

      , 로 변환
숫자 1번 키 왼쪽에 있는 `(Grave) 입력

**인라인(inline) 코드 강조**
`background`혹은 `background-image` 속성으로 요소에 배경 이미지를 삽입 가능

**블록(block) 코드 강조**
`를 3번 이상 입력하고 코드 종류도 입력











### 표(Table)

 태그로 변환
헤더 셀을 구분할 때 3개 이상의 -(hyphen/dash) 기호 vlfdy
헤더 셀을 구분하면서 :(Colons) 기호로 셀(열/칸) 안에 내용 정렬 가능
가장 좌측과 가장 우측에 있는 |(vertical bar) 기호는 생략 가능

| 값 | 의미 | 기본값 |
|---|:---:|---:|
| `static` | 유형(기준) 없음 / 배치 불가능 | `static` |
| `relative` | 요소 자신을 기준으로 배치 |  |
| `absolute` | 위치 상 부모(조상)요소를 기준으로 배치 |  |
| `fixed` | 브라우저 창을 기준으로 배치 |  |

값 | 의미 | 기본값
---|:---:|---:
`static` | 유형(기준) 없음 / 배치 불가능 | `static`
`relative` | 요소 **자신**을 기준으로 배치 |
`absolute` | 위치 상 **_부모_(조상)요소**를 기준으로 배치 |
`fixed` | **브라우저 창**을 기준으로 배치 |

>>> 중중첩된 인용문 2
>>> 중중첩된 인용문 3

                            201018 REVIEW - How to start git
                        
                    
                
      
                
      
    
	
		    Oct 19, 2020
    	
    
    

      

            
      
            
                
      
                    
      [How to start git]
      https://ulgoon.github.io/2019/09/09/01-git-first/ 
      

      1. git init
      (1)    프로젝트 폴더 만들기
          -    $ mkdir sample
          -    $ cd sample
      (2)    git init 명령으로 local repository 역할 시작
          -    $ git init  (= 이제부터 git을 쓰겠다)
      (3)    github에서 새로운 remote repository를 생성한 뒤, 복사한 주소를 local repository에 등록
          -    remote repository 이름은 local repository 이름과 맞춰서 헷갈리지 않도록 함
      https://github.com/green9930/sample.git
          -    주소를 복사해 현재 작업공간에 remote repository 주소를 추가
              $ git remote add origin https://github.com/green9930/sample.git
      

      **origin 은 복사한 주소를 부르기 위한 이름이며, 다른 이름을 써도 무방
          -    주소가 잘 추가되었는지 확인
              $ git remote get-url origin
      

      (4)    새 파일을 만들고 작업을 수행
          -    $ touch README.md
          -    $ vi README.md
                  i(insert mode) 텍스트 편집
                  esc(normal mode)
                  :wq : 저장한 뒤 종료
      (5)    작업이 끝났으면 $ git status 명령으로 현재 프로젝트의 상태 확인
          -> untracked files: README.md 가 보인다면 다음 작업 준비가 끝났다는 의미
      (6)    workspace에 추가된 새 파일을 index로 staging
          -    $ git add README.md
      (7)    추가된 README.md에 대한 설명을 commit
          -    $ git commit
                  vim 에디터로 commit message 작성
                  $ git commit -m “ commit message “
      

          **_commit message 깔끔하게 입력하기_**
        open source의 경우 모든 사용자가 읽을 수 있는 언어 선택
        commit message의 제목에 말머리를 달아 commit의 카테고리를 바로 확인할 수 있게 함
        feat:, docs:, test:, refactor: 등
        Referance: Conventional commits
        commit message의 제목은 구의 형태를 띄게 함
        commit message는 제목과 내용이 존재
        내용은 optional body에 Enter로 구분하여 작성하며, 문장형으로 작성
      

      (8)     commit이 끝났으면 remote repository로 push
          -    첫 commit이면 -u(-set-upstream) flag를 붙여 push하여 현재 작업중인 branch가 업로드할 branch의 변동사항을 추적하도록 설정
              $ git push -u origin master
          -    이후에는 -u 옵션 없이 바로 push
              $ git push origin master
      

      2. git clone  / github에서 remote repository를 생성한 뒤, 작업공간으로 끌어와 작업 시작
      (1)    github에서 remote repository를 생성하되, LICENSE, README.md, .gitignore 등의 파일과 함께 생성하고 해당 repository 주소 복사
      (2)    git bash에서 $ git clone을 복사한 주소를 입력해 작업 공간에 local repository 생성
      (3)    새 파일을 만들고 작업 수행
          -    touch README.md
          -    vi README.md
      (4)    작업이 끝나면 $ git status 명령으로 현재 프로젝트의 상태 확인
          -    untracked files: README.md -> 다음 작업을 준비가 된 상태
      (5)    workspace에 추가된 새로운 파일을 index로 staging
          -    $ git add README.md
      (6)    추가된 README.md에 대한 설명을 commit
          -    $ git commit’
      (7)    commit이 끝났다면 remote repository로 push
